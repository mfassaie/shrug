## Overview

A dynamic CLI for Atlassian Cloud — commands generated at runtime from OpenAPI specs, supporting Jira, Jira Software, Confluence, BitBucket, and Service Management. Built in Rust, inspired by Google Workspace CLI architecture.

## Phase Details

### Phase 1: Project Foundation (3 plans)
- 01-01: Cargo project scaffold, dependencies, and project structure
- 01-02: Config system with TOML, layered precedence, and platform paths
- 01-03: Enhanced logging, Ctrl+C handling, and config debug dump

### Phase 2: OpenAPI Spec Engine (4 plans)
- 02-01: OpenAPI 3.0.1 spec parser and data model
- 02-02: Swagger 2.0 parser / conversion layer for BitBucket
- 02-03: Spec caching (JSON, bundled fallback, version detection)
- 02-04: Spec analysis & conformance test suite

### Phase 3: Dynamic Command Tree (2 plans)
- 03-01: Two-phase parsing and product routing
- 03-02: Command tree builder (tags, operations, parameters, help)

### Phase 4: Authentication & Profiles (3 plans)
- 04-01: Profile management and config integration
- 04-02: Keychain credential storage with encrypted file fallback
- 04-03: OAuth 2.0 flow, token refresh, and interactive setup wizard

### Phase 5: Generic HTTP Executor (4 plans)
- 05-01: URL building, request construction, and basic execution
- 05-02: Rate limiting, retries, and error response handling
- 05-03: Unified pagination iterator (offset, cursor, link-based)
- 05-04: Quirks registry (operationId → special headers)

### Phase 6: Output & Formatting (2 plans)
- 06-01: Output formatters (table, JSON, YAML, CSV, plain) with TTY detection
- 06-02: ADF terminal rendering, pager integration, and --fields selection

### Phase 7: Helper Commands & ADF (3 plans)
- 07-01: Markdown → ADF converter and JQL shorthand flags
- 07-02: Helper commands (+create, +search, +transition)
- 07-03: Shell completions and field/user resolution caches

### Phase 8: Distribution & Polish (3 plans)
- 08-01: cargo-dist release pipeline, Homebrew tap, and Scoop manifest
- 08-02: Mock-based integration tests with recorded API responses
- 08-03: E2E smoke tests, performance benchmarks, and first-run polish
