# Roadmap: shrug

## Overview

A dynamic CLI for Atlassian Cloud — commands generated at runtime from OpenAPI specs, supporting Jira, Jira Software, and Confluence. Built in Rust, inspired by Google Workspace CLI architecture.

## Phase Details

### Phase 21: Installed Binary Test Harness

**Focus:** New E2E harness that finds shrug.exe on PATH or via SHRUG_E2E_BINARY env var
**Plans:** TBD

**Scope:**
- New test harness finding shrug.exe on PATH instead of cargo_bin
- SHRUG_E2E_BINARY env var override for explicit binary path
- RunResult with exit code, stdout, stderr assertions
- Offline tests run without Atlassian credentials
- Online tests gated by skip_unless_e2e! macro with existing E2E credentials
- Reuse E2eConfig for live API tests

**Constraints:**
- Must coexist with existing cargo_bin E2E tests
- Must work on Windows (primary target)

### Phase 22: Static Commands & Global Flags

**Focus:** Exercise every non-API command and global flag combination against installed binary
**Plans:** TBD

**Scope:**
- --version, --help output validation
- auth status, auth setup (non-interactive check)
- profile list/create/show/delete/use
- cache refresh (dry-run or offline)
- completions powershell/bash/zsh/fish
- Global flag matrix: --output (json/table/yaml/csv/plain), --color (auto/always/never), --dry-run, -v/-vv/--trace, --no-pager, --fields

**Constraints:**
- All offline — no Atlassian credentials required
- Tests must pass on clean Windows install

### Phase 23: Help & Error Message Snapshots

**Focus:** insta golden-file tests for all help output plus error message validation
**Plans:** TBD

**Scope:**
- insta golden-file snapshots for all --help output (top-level, auth, profile, cache, completions, product-level help)
- Error message + remediation hint validation for each ShrugError variant (bad token, missing profile, bad usage, not found, etc.)
- Exit code verification per error type

**Constraints:**
- insta crate (already a dependency) used for snapshot testing
- Snapshots must be platform-stable (no Windows-specific path differences in output)

### Phase 24: Live API CRUD Smoke Tests

**Focus:** Basic Jira and Confluence CRUD against live Atlassian Cloud via installed binary
**Plans:** TBD

**Scope:**
- Jira: create issue → verify → delete (cleanup)
- Confluence: create page → verify → delete (cleanup)
- Uses existing E2E credentials (SHRUG_E2E_SITE, SHRUG_E2E_EMAIL, SHRUG_E2E_TOKEN)
- ResourceTracker pattern for cleanup on failure
- Rate limit delay between operations

**Constraints:**
- Gated behind E2E env vars — skipped when not configured
- Must clean up all created resources even on test failure
