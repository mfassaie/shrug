<objective>
## Goal
Build a command tree display that shows available operations organized by tag, with help text from spec descriptions, parameter info, and HTTP method/path details. Wire into main.rs so `shrug jira issues` shows a rich operation list.

## Purpose
Users need to discover available API operations without reading Atlassian docs. The command tree provides self-documenting CLI help powered by the spec, showing what operations exist, their parameters, and HTTP details.

## Output
- `src/cmd/tree.rs` — Command tree builder and display formatter
- Updated `src/cmd/mod.rs` — module wiring
- Updated `src/main.rs` — use tree display for tag-only and help scenarios
</objective>


## AC-3: Operation detail display
```gherkin
Given a resolved operation
When the command resolves successfully (before Phase 5 HTTP execution)
```

## AC-4: Parameter display for operations
```gherkin

```

<task type="auto">
  <name>Task 1: Command tree display builder</name>
  <files>src/cmd/tree.rs, src/cmd/mod.rs</files>
  <action>
    Create src/cmd/tree.rs with display formatting functions:

    1. **format_tag_list(spec: &ApiSpec) -> String:**
       - Show each tag with description and operation count
       - Format:
         ```
         Available command groups:
           issues         Issue operations (5 operations)
           projects       Project operations (3 operations)
         ```

    2. **format_operations(spec: &ApiSpec, tag: &str) -> String:**
       - List operations for a tag with command name, method, and summary
       - Mark deprecated operations with [deprecated]
       - Format:
         ```
         Operations for 'issues':
           create-issue                  POST   Create an issue
           get-issue                     GET    Get an issue
           edit-issue [deprecated]       PUT    Edit an issue
           search-for-issues-using-jql   GET    Search for issues using JQL
         ```

    3. **format_operation_detail(operation: &Operation, server_url: Option<&str>) -> String:**
       - Show full operation details including parameters
       - Format:
         ```
         get-issue — Get an issue
           GET /rest/api/3/issue/{issueIdOrKey}
           Server: https://example.atlassian.net

           Parameters:
             issueIdOrKey  path   required  string  The ID or key of the issue
             expand        query  optional  string  Fields to expand
             fields        query  optional  string  Fields to return
         ```

    4. **format_params(params: &[Parameter]) -> String:**
       - Format parameter table with columns: name, location, required/optional, type, description
       - Required params listed first

    Wire in src/cmd/mod.rs:
    - Add `pub mod tree;`

    Include unit tests:
    - format_tag_list with sample spec
    - format_operations shows command names and methods
    - format_operations marks deprecated
    - format_operation_detail shows parameters
    - format_params sorts required first
