<objective>
## Goal
Add dynamic tab-completion for project keys, space keys, and issue keys from live Atlassian instances. Results are cached locally with a short TTL for fast interactive completion.

## Purpose
Shell completions for resource identifiers remove friction. Users tab-complete `--project F<TAB>` to get `FOO` without memorising keys.

## Output
- `src/dynamic_completions.rs` — fetch + cache completion values from Atlassian APIs
- Hidden `_complete` CLI subcommand that outputs completion values
- Updated `completions` command to generate dynamic-aware shell scripts
</objective>

<task type="auto">
  <name>Task 1: Dynamic completions core (fetch, cache, _complete subcommand)</name>
  <files>src/dynamic_completions.rs, src/cli.rs, src/main.rs, src/lib.rs</files>
  <action>
    1. Create `src/dynamic_completions.rs`:

       a. `CompletionCache` struct wrapping a cache directory:
          - Cache file per type: `completions/{type}.json`
          - JSON format: `{"values": ["KEY1","KEY2"], "cached_at": "ISO timestamp"}`
          - TTL: 300 seconds (5 minutes)
          - `load(type) -> Option<Vec<String>>` (returns None if expired or missing)
          - `save(type, values)` (atomic write)

       b. `fetch_projects(client, credential) -> Result<Vec<String>>`:
          - GET /rest/api/3/project (Jira)
          - Extract `key` from each project
          - Return sorted list

       c. `fetch_spaces(client, credential) -> Result<Vec<String>>`:
          - GET /wiki/api/v2/spaces?limit=250
          - Extract `key` from each space
          - Return sorted list

       d. `fetch_issues(client, credential, project) -> Result<Vec<String>>`:
          - GET /rest/api/3/search?jql=project={project}&fields=key&maxResults=50
          - Extract `key` from each issue
          - Return sorted list

       e. `complete(type, client, credential, cache, extra_args) -> Vec<String>`:
          - Check cache first
          - If miss: fetch from API, save to cache
          - Return values (or empty vec on error, never fail for completions)

    2. Add `pub mod dynamic_completions;` to lib.rs.

    3. Add hidden `_complete` subcommand to CLI:
       - In cli.rs Commands enum:
         ```rust
         #[command(name = "_complete", hide = true)]
         Complete {
             /// Completion type: projects, spaces, issues
             completion_type: String,
             /// Extra args (e.g. --project FOO for issue completion)
             #[arg(trailing_var_arg = true)]
             args: Vec<String>,
         },
         ```

    4. Handle `_complete` in main.rs:
       - Create reqwest client + resolve credential
       - Create CompletionCache from cache dir
       - Call `complete(type, ...)` and print each value on its own line
       - Exit immediately (fast path, minimal overhead)

    5. Add unit tests for CompletionCache (save/load/TTL).
  </action>
  <verify>`cargo build` and `cargo test` pass</verify>
  <done>AC-1 and AC-2 satisfied</done>
</task>
