<task type="auto">
  <name>Task 1: Issue CRUD test and Jira test module setup</name>
  <files>tests/e2e/jira.rs, tests/e2e/main.rs</files>
  <action>
    **tests/e2e/main.rs:** Add `mod jira;`

    **tests/e2e/jira.rs:**

    Add helper to create a temp profile + set as default (reuse pattern from auth.rs):
    ```
    fn setup_profile(runner: &ShrugRunner) -> String  // returns profile name
    fn teardown_profile(runner: &ShrugRunner, name: &str)
    ```

    **test_issue_crud_lifecycle():**
    - Setup: create temp profile, set as default
    - CREATE: `shrug --output json jira Issues create-issue --json '{"fields":{"project":{"key":"{project}"},"summary":"E2E CRUD test issue","issuetype":{"name":"Task"}}}'`
      - Assert exit 0, parse JSON, extract issue key
    - READ: `shrug --output json jira Issues get-issue --issueIdOrKey {key}`
      - Assert exit 0, response contains the summary
    - UPDATE: `shrug jira Issues edit-issue --issueIdOrKey {key} --json '{"fields":{"summary":"E2E CRUD updated"}}'`
      - Assert exit 0 (204 No Content is normal for edit)
    - READ again: verify summary is now "E2E CRUD updated"
    - DELETE: `shrug jira Issues delete-issue --issueIdOrKey {key}`
      - Assert exit 0
    - Teardown: delete temp profile
    - Call rate_limit_delay() between API calls

    Avoid: Hardcoding issue type names — use "Task" which exists in all Jira projects.
    Avoid: Creating issues without cleanup — always delete, even on assertion failure (use ResourceTracker or manual cleanup in a defer-style pattern).
  </action>
  <verify>
    `cargo test --test e2e jira::test_issue_crud -- --test-threads=1 --nocapture` passes.
  </verify>
  <done>AC-1 satisfied: full issue CRUD lifecycle test passes against live Cloud</done>
</task>

<task type="auto">
  <name>Task 2: Comment and Worklog CRUD tests</name>
  <files>tests/e2e/jira.rs</files>
  <action>
    **test_comment_crud_lifecycle():**
    - Setup: create temp profile, create test issue (parent)
    - ADD: `shrug --output json jira "Issue comments" add-comment --issueIdOrKey {key} --json '{"body":{"type":"doc","version":1,"content":[{"type":"paragraph","content":[{"type":"text","text":"E2E test comment"}]}]}}'`
      - Assert exit 0, extract comment ID from response
    - READ: `shrug --output json jira "Issue comments" get-comment --issueIdOrKey {key} --id {commentId}`
      - Assert exit 0, response contains the comment
    - UPDATE: `shrug jira "Issue comments" update-comment --issueIdOrKey {key} --id {commentId} --json '{"body":{"type":"doc","version":1,"content":[{"type":"paragraph","content":[{"type":"text","text":"E2E updated comment"}]}]}}'`
      - Assert exit 0
    - DELETE: `shrug jira "Issue comments" delete-comment --issueIdOrKey {key} --id {commentId}`
      - Assert exit 0
    - Teardown: delete test issue, delete profile
    - rate_limit_delay() between calls

    **test_worklog_crud_lifecycle():**
    - Setup: create temp profile, create test issue
    - ADD: `shrug --output json jira "Issue worklogs" add-worklog --issueIdOrKey {key} --json '{"timeSpentSeconds":3600,"comment":{"type":"doc","version":1,"content":[{"type":"paragraph","content":[{"type":"text","text":"E2E worklog"}]}]}}'`
      - Assert exit 0, extract worklog ID
    - READ: `shrug --output json jira "Issue worklogs" get-worklog --issueIdOrKey {key} --id {worklogId}`
      - Assert exit 0
    - UPDATE: `shrug jira "Issue worklogs" update-worklog --issueIdOrKey {key} --id {worklogId} --json '{"timeSpentSeconds":7200}'`
      - Assert exit 0
    - DELETE: `shrug jira "Issue worklogs" delete-worklog --issueIdOrKey {key} --id {worklogId}`
      - Assert exit 0
    - Teardown: delete test issue, delete profile
    - rate_limit_delay() between calls

    Avoid: Using plain text for comment/worklog bodies — Jira Cloud requires ADF format.
    Avoid: Leaving test issues behind if comment/worklog operations fail.
  </action>
