<objective>
## Goal
Implement profile management with TOML-based storage, CRUD operations (create, list, show, delete, use), and CLI subcommands. Profiles store site URL, email, and auth type — credentials are NOT stored here (that's Plan 04-02). Default profile tracked via single `.default` file, not per-profile flags.

## Purpose
Profiles are the foundation of multi-site authentication. Users need to configure which Atlassian site they're targeting before any API calls can be made. This plan delivers the profile lifecycle; credential storage comes in 04-02.

<acceptance_criteria>

## AC-1: Profile creation
```gherkin
Given no profiles exist
When user runs `shrug profile create --name work --site mysite.atlassian.net --email user@example.com`
Then a profile "work" is saved to disk as TOML
And site URL is normalized to include https:// prefix if missing
And the profile is set as default (first profile auto-defaults)
```

## AC-2: Profile listing
```gherkin
Given one or more profiles exist
When user runs `shrug profile list`
Then all profiles are displayed with name, site, email, and default marker
And the default profile is marked with an asterisk (*)
```

## AC-3: Profile show
```gherkin
Given a profile "work" exists
When user runs `shrug profile show --name work`
Then the profile details are displayed (name, site, email, auth_type, whether it is the default)
```

## AC-3b: Profile not found <!-- audit-added -->
```gherkin
Given no profile named "nonexistent" exists
When user runs `shrug profile show --name nonexistent`
Then a NotFound error is returned with message "Profile 'nonexistent' not found"
And exit code is 4 (NOT_FOUND)
```

## AC-4: Profile deletion
```gherkin
Given a profile "work" exists
When user runs `shrug profile delete --name work`
Then the profile is removed from storage
And if it was the default, the .default file is also removed (user must set a new default)
```

## AC-4b: Delete non-existent profile <!-- audit-added -->
```gherkin
Given no profile named "nonexistent" exists
When user runs `shrug profile delete --name nonexistent`
Then a NotFound error is returned with message "Profile 'nonexistent' not found"
And exit code is 4 (NOT_FOUND)
```

## AC-5: Profile use (set default)
```gherkin
Given profiles "work" and "personal" exist
When user runs `shrug profile use --name personal`
Then "personal" becomes the default profile
And "work" is no longer marked as default
```

## AC-6: Profile resolution from config
```gherkin
Given a default profile exists
When any product command runs (e.g., shrug jira ...)
Then the active profile is resolved via: --profile flag > SHRUG_PROFILE env > config default_profile
And the profile's site URL is available for HTTP executor (Phase 5)
```

## AC-7: Profile validation
```gherkin
Given a user attempts to create a profile
When the site URL or email is invalid or missing
Then a clear error message is returned
And no profile file is written
```

    1. Create `src/auth/mod.rs` with `pub mod profile;`
    2. Add `pub mod auth;` to `src/lib.rs`
    3. Create `src/auth/profile.rs` with:

    **Profile struct:**
    - `name: String` — unique identifier (lowercase, alphanumeric + hyphen)
    - `site: String` — Atlassian site URL (e.g., "https://mysite.atlassian.net")
    - `email: String` — user email for Basic Auth
    - `auth_type: AuthType` — enum: BasicAuth, OAuth2 (default BasicAuth)
    - NO `is_default` field — default tracked via `.default` file (see below) <!-- audit-added -->
    - Derive: Debug, Clone, Serialize, Deserialize

    **AuthType enum:**
    - `BasicAuth` — email + API token
    - `OAuth2` — OAuth 2.0 3LO flow
    - Derive: Debug, Clone, PartialEq, Serialize, Deserialize, Default (BasicAuth)

    **ProfileStore struct:**
    - `profiles_dir: PathBuf` — directory where profile TOML files live
    - Constructor: `new(profiles_dir: PathBuf) -> Result<Self, ShrugError>` — creates dir if missing
    - `create(&self, profile: &Profile) -> Result<(), ShrugError>` — validates, writes TOML, auto-defaults if first
    - `list(&self) -> Result<Vec<Profile>, ShrugError>` — reads all .toml files in profiles_dir
    - `get(&self, name: &str) -> Result<Profile, ShrugError>` — reads single profile
    - `delete(&self, name: &str) -> Result<(), ShrugError>` — removes file
    - `set_default(&self, name: &str) -> Result<(), ShrugError>` — validates profile exists, writes name to `.default` file (single atomic write, no multi-file update) <!-- audit-added -->
    - `get_default(&self) -> Result<Option<Profile>, ShrugError>` — reads `.default` file, loads that profile
    - `clear_default(&self) -> Result<(), ShrugError>` — removes `.default` file <!-- audit-added -->
    - Private: `profile_path(&self, name: &str) -> PathBuf` — profiles_dir.join("{name}.toml")
    - Private: `default_path(&self) -> PathBuf` — profiles_dir.join(".default") <!-- audit-added -->
    - Private: `validate_name(name: &str) -> Result<(), ShrugError>` — lowercase alphanumeric + hyphens, 1-64 chars
    - Private: `normalize_site(site: &str) -> String` — ensure https:// prefix, strip trailing slash

    **Storage format:** One TOML file per profile at `{config_dir}/profiles/{name}.toml`
    **Default tracking:** Single file `{config_dir}/profiles/.default` containing just the profile name (plain text). This avoids multi-file atomicity issues with `is_default` flags. <!-- audit-added -->

    **Validation:**
    - Name: `^[a-z0-9][a-z0-9-]{0,63}$` (regex pattern, validated manually without regex crate)
    - Site: must contain at least one dot, normalized to https://
    - Email: must contain `@`
    - Duplicate name: error on create if file exists

    4. Add `ProfileError(String)` variant to ShrugError in `src/error.rs` with exit code GENERAL_ERROR. Use `ShrugError::NotFound` for profile-not-found cases (get, delete, show, use on non-existent name) — do NOT use ProfileError for not-found. <!-- audit-added -->
    5. No new crate dependencies needed — serde + toml already available
    6. **Atomic file writes:** All profile TOML writes (create, update) must use write-to-temp-then-rename pattern: write to `{name}.toml.tmp` in same directory, then `fs::rename` to `{name}.toml`. Prevents partial writes from corrupting profile data. <!-- audit-added -->
    7. **Graceful list() on corrupted files:** `list()` must catch per-file deserialization errors, log a warning via `tracing::warn!`, and skip corrupted files rather than failing the entire operation. <!-- audit-added -->

    **Tests (in profile.rs):**
    - Profile creation writes valid TOML file
    - Profile list returns all profiles
    - Profile get by name
    - Profile get non-existent returns NotFound <!-- audit-added -->
    - Profile delete removes file
    - Profile delete non-existent returns NotFound <!-- audit-added -->
    - Set default writes .default file with profile name <!-- audit-added -->
    - Set default for non-existent profile returns NotFound <!-- audit-added -->
    - Get default reads from .default file <!-- audit-added -->
    - First profile auto-defaults (writes .default file) <!-- audit-added -->
    - Delete default profile also removes .default file <!-- audit-added -->
    - Invalid name rejected (spaces, uppercase, too long)
    - Invalid site rejected (no dots)
    - Invalid email rejected (no @)
    - Duplicate name returns error
    - Normalize site adds https:// prefix
    - Normalize site strips trailing slash
    - List skips corrupted TOML files with warning <!-- audit-added -->
