<objective>
## Goal
Implement credential storage with OS keychain as primary backend and encrypted file as fallback. Credentials (API tokens) are stored and retrieved by profile name. Environment variable override supported for CI/CD.

## Purpose
Profiles (04-01) store metadata but no secrets. This plan adds secure credential storage so that Phase 5's HTTP executor can inject auth headers. The keychain-first approach meets the compliance constraint that credentials must never be stored in plaintext.

## Output
- `src/auth/credentials.rs` — CredentialStore trait, KeychainBackend, EncryptedFileBackend, EnvBackend
- Updated `src/auth/mod.rs` — Module declaration
- Updated `src/main.rs` — Credential store wired into profile commands (store/delete token on profile create/delete)
- Updated `Cargo.toml` — keyring, aes-gcm, argon2, base64, rand dependencies
</objective>

<acceptance_criteria>

## AC-1: Store API token via keychain
```gherkin
Given a profile "work" exists
When user runs `shrug auth set-token --profile work`
Then user is prompted for their API token (hidden input)
And the token is stored in the OS keychain under service "shrug" with key "work"
And confirmation message is displayed
```

## AC-2: Retrieve credential for profile
```gherkin
Given profile "work" has a stored API token
When the credential store resolves credentials for "work"
Then the API token is returned along with the profile's email
And the result includes the auth type (BasicAuth)
```

## AC-3: Encrypted file fallback
```gherkin
Given the OS keychain is not available (e.g., headless server, CI container)
When user stores a credential
Then the credential is encrypted with AES-256-GCM using a key derived from a user-provided password (argon2)
And the encrypted blob is stored at {data_dir}/credentials/{profile-name}.enc
And retrieval prompts for the password to decrypt
```

## AC-4: Environment variable override
```gherkin
Given SHRUG_API_TOKEN is set in the environment
When credential resolution runs for any profile
Then the env var token is used instead of keychain/file lookup
And SHRUG_EMAIL overrides profile email if set
And SHRUG_SITE overrides profile site if set
```

## AC-5: Delete credentials with profile
```gherkin
Given profile "work" has stored credentials (keychain or file)
When user runs `shrug profile delete --name work`
Then the credentials are also removed from keychain/file
And a warning is shown if credential deletion fails (non-blocking)
```

## AC-6: Credential status display
```gherkin
Given profile "work" exists
When user runs `shrug profile show --name work`
Then the output includes whether credentials are stored (token: set/not set)
And does NOT display the actual token value
```

## AC-7: Keychain unavailable detection
```gherkin
Given the OS keychain backend fails (permission denied, not installed)
When the credential store initializes
Then it falls back to encrypted file backend
And logs a debug message explaining the fallback
```

    **ResolvedCredential struct:**
    - `email: String` — from profile or SHRUG_EMAIL env
    - `api_token: String` — from keychain/file/env
    - `site: String` — from profile or SHRUG_SITE env
    - `source: CredentialSource` — enum: Keychain, EncryptedFile, Environment

    **CredentialSource enum:**
    - `Keychain` — from OS keychain
    - `EncryptedFile` — from encrypted file fallback
    - `Environment` — from env vars
    - Derive: Debug, Clone, PartialEq

    **CredentialStore struct:**
    - `data_dir: PathBuf` — for encrypted file fallback path
    - `backend: CredentialBackendType` — enum dispatch (KeychainBackend or EncryptedFileBackend), not trait object. Simpler, avoids dyn dispatch complexity for 2 concrete types. <!-- audit-modified -->
    - Constructor: `new(data_dir: PathBuf) -> Self` — creates with keychain as primary, lazy fallback to encrypted file on first error
    - `store(&self, profile_name: &str, token: &str) -> Result<(), ShrugError>` — store token
    - `retrieve(&self, profile_name: &str) -> Result<Option<Credential>, ShrugError>` — get token
    - `delete(&self, profile_name: &str) -> Result<(), ShrugError>` — remove token
    - `has_credential(&self, profile_name: &str) -> bool` — check existence without retrieving
    - `resolve(profile: &Profile, data_dir: &Path) -> Result<Option<ResolvedCredential>, ShrugError>` — full resolution: env vars first, then backend lookup

    **CredentialBackend trait:** <!-- audit-modified: passwords/tokens passed as parameters, not prompted inside backends -->
    - `store(&self, profile_name: &str, token: &str) -> Result<(), ShrugError>`
    - `retrieve(&self, profile_name: &str) -> Result<Option<String>, ShrugError>` — for EncryptedFileBackend, takes password param (see below)
    - `delete(&self, profile_name: &str) -> Result<(), ShrugError>`
    - `has_credential(&self, profile_name: &str) -> Result<bool, ShrugError>` <!-- audit-added: Result instead of bool to surface keychain errors -->
    - `source(&self) -> CredentialSource`

    **IMPORTANT (audit-added):** All interactive I/O (rpassword prompts) MUST happen at the CLI layer (main.rs), NOT inside backend implementations. Backends receive passwords/tokens as function parameters. This enables testing, non-interactive use, and clean separation of concerns.

    **KeychainBackend:**
    - Uses `keyring::Entry::new("shrug", profile_name)` for each credential
    - `store`: `entry.set_password(token)`
    - `retrieve`: `entry.get_password()` → `Ok(Some(token))` or `Ok(None)` if not found
    - `delete`: `entry.delete_credential()`
    - `has_credential`: try `get_password`, return `Ok(true)` if found, `Ok(false)` if NoEntry, `Err` for real failures <!-- audit-added -->
    - Handle keyring errors by mapping to `ShrugError::AuthError`
    - Constructor: `new() -> Self` — NO keychain probe. Attempt keychain on first real use and fall back lazily. Store a `fallback_triggered: Cell<bool>` flag so subsequent calls go directly to fallback if first attempt failed. <!-- audit-modified: no probe entry, lazy fallback -->

    **EncryptedFileBackend:** <!-- audit-modified: no rpassword calls inside backend -->
    - `credentials_dir: PathBuf` — `{data_dir}/credentials/`
    - Constructor: `new(data_dir: &Path) -> Result<Self, ShrugError>` — creates dir if missing
    - `store_with_password(&self, profile_name: &str, token: &str, password: &str) -> Result<(), ShrugError>` — derive key with argon2 (random 16-byte salt), encrypt token with AES-256-GCM (random 12-byte nonce), store as base64-encoded JSON: `{"salt": "...", "nonce": "...", "ciphertext": "..."}` at `{credentials_dir}/{profile_name}.enc`
    - `retrieve_with_password(&self, profile_name: &str, password: &str) -> Result<Option<String>, ShrugError>` — read .enc file, derive key from salt, decrypt ciphertext
    - `delete`: remove .enc file
    - `has_credential`: return `Ok(true)` if .enc file exists, `Ok(false)` otherwise <!-- audit-modified -->
    - Use atomic write pattern (temp-then-rename) from 04-01
    - **NOTE:** rpassword prompting happens in main.rs, NOT here. Backend receives password as parameter. This enables unit testing with known passwords. <!-- audit-added -->

    **Environment resolution (in resolve()):**
    - Check `SHRUG_API_TOKEN` env var — if set, use it directly (source: Environment)
    - Check `SHRUG_EMAIL` env var — if set, override profile email
    - Check `SHRUG_SITE` env var — if set, override profile site
    - If env var provides token, skip keychain/file lookup entirely

    4. Add `CredentialError(String)` variant to ShrugError if needed, or reuse `AuthError` for credential failures.

    **Tests (in credentials.rs):** <!-- audit-modified: tests use password parameter directly, no rpassword mocking needed -->

    - EncryptedFileBackend delete removes file
    - EncryptedFileBackend has_credential checks file existence
    - EncryptedFileBackend wrong password returns error (decryption failure)
    - EncryptedFileBackend corrupted .enc file returns clear error
    - Environment resolution returns env vars when set
    - Environment resolution skips backend when SHRUG_API_TOKEN set

    - has_credential returns Result<bool> not plain bool <!-- audit-added -->
