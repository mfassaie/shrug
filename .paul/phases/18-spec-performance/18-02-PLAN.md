<objective>
## Goal
Add ETag-based conditional fetch and serve-stale with background refresh. When a cached spec has expired, return it immediately (stale) and spawn a background thread to refresh via conditional GET. Subsequent invocations get the updated cache.

## Purpose
Eliminates perceptible startup delay when cache expires. Users never wait for a spec fetch. The CLI feels instant on every invocation while keeping specs current.

## Output
- ETag stored in CacheMetadata on fetch, sent as If-None-Match on refresh
- 304 Not Modified handled (touch TTL without re-parsing)
- Serve-stale pattern: expired cache returned immediately, background thread refreshes
- Background thread uses conditional fetch, updates both JSON and binary cache
</objective>

<task type="auto">
  <name>Task 1: ETag storage, conditional fetch, and TTL touch</name>
  <files>src/spec/cache.rs, src/spec/registry.rs</files>
  <action>
    **cache.rs changes:**

    1. Add `save_with_etag(&self, product: &str, spec: &ApiSpec, etag: Option<String>)` method:
       - Same as `save()` but stores the provided ETag in CacheMetadata
       - Also dual-writes binary cache (same as save)
       - Update `save()` to delegate to `save_with_etag(product, spec, None)`

    2. Add `load_etag(&self, product: &str) -> Result<Option<String>, ShrugError>`:
       - Read JSON cache entry metadata, return etag field
       - Returns None if no cache or no ETag stored

    3. Add `touch_ttl(&self, product: &str) -> Result<(), ShrugError>`:
       - Load JSON cache entry
       - Update `cached_at` to `Utc::now()`
       - Write back (atomic write pattern)
       - Does NOT touch binary cache (TTL is tracked via JSON metadata only)

    4. Add unit tests:
       - `save_with_etag_stores_etag`: save with ETag, verify load_etag returns it
       - `save_without_etag_stores_none`: save without ETag, verify load_etag returns None
       - `touch_ttl_refreshes_timestamp`: save with old timestamp, touch_ttl, verify load succeeds with fresh TTL

    **registry.rs changes:**

    5. Update `fetch_spec()` to capture ETag:
       - After successful response, extract `ETag` header value
       - Call `self.cache.save_with_etag(key, &spec, etag)` instead of `self.cache.save(key, &spec)`

    6. Add `conditional_fetch(&self, product: &Product) -> Result<ConditionalFetchResult, ShrugError>`:
       - Check `self.cache.load_etag(cache_key)` for stored ETag
       - If ETag exists: send GET with `If-None-Match: {etag}` header
       - If 304: call `self.cache.touch_ttl(cache_key)`, return `NotModified`
       - If 200: parse spec, save with new ETag, return `Updated(ApiSpec)`
       - If no ETag stored: do normal fetch, return `Updated(ApiSpec)`
       - Define enum: `enum ConditionalFetchResult { NotModified, Updated(ApiSpec) }`

    7. Update existing `fetch_spec()` to use the ETag-aware save path.

<task type="auto">
  <name>Task 2: Serve-stale with background refresh thread</name>
  <files>src/spec/registry.rs, src/main.rs</files>
  <action>
    **registry.rs changes:**

    1. Update `SpecLoader::load()` to implement serve-stale:
       - Try cache with TTL (fresh): return immediately (unchanged)
       - If cache expired: try `load_stale()`. If stale exists:
         - Clone necessary info (cache_dir path, product info, ETag)
         - Spawn `std::thread::spawn` with a closure that runs `background_refresh()`
         - Return stale spec immediately (don't wait for thread)
       - If no cache at all: synchronous fetch (unchanged fallback to network → bundled)

    2. Add standalone function `background_refresh(cache_dir: PathBuf, spec_url: String, cache_key: String, spec_format: SpecFormat, etag: Option<String>)`:
       - Creates its own `SpecCache` from cache_dir
       - Creates its own `reqwest::blocking::Client`
       - If ETag present: conditional GET with If-None-Match
         - 304: call `cache.touch_ttl(cache_key)`
         - 200: parse spec, `cache.save_with_etag(cache_key, &spec, new_etag)`
       - If no ETag: normal GET, save with ETag
       - Log all outcomes via tracing (debug level)
       - Catch and log all errors (non-fatal, best-effort refresh)
       - The parse_spec call needs to handle both V3 and V2 formats, use the spec_format parameter

    3. The background thread is detached (JoinHandle dropped). If the process exits before completion, the cache stays in its current consistent state (atomic writes protect against corruption). Next invocation retries.

    **main.rs changes:**

    4. No changes needed. `handle_product()` and helper dispatch both call `loader.load(&product)` which now implements serve-stale internally.

    5. Add unit tests in registry.rs:
       - `load_returns_stale_when_expired`: verify expired cache is served (not None)
       - Test that fresh cache is returned without triggering background work
  </action>
