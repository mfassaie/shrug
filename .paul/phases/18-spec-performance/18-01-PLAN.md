<objective>
## Goal
Add rkyv zero-copy binary spec cache alongside the existing JSON cache. Binary cache is preferred on read; JSON is fallback. Target: <0.5ms warm load (down from ~5ms JSON parse for 2.5MB Jira spec).

## Purpose
Spec loading is the primary startup cost. rkyv zero-copy deserialisation eliminates the parsing step entirely for warm starts, making shrug feel instant.

## Output
- rkyv dependency added to Cargo.toml
- Archive/Serialize/Deserialize derives on all spec model types
- Binary cache save/load in SpecCache (alongside existing JSON)
- Benchmark test comparing JSON vs rkyv load times
</objective>

<task type="auto">
  <name>Task 2: Binary cache save/load in SpecCache</name>
  <files>src/spec/cache.rs</files>
  <action>
    1. Add binary save method to SpecCache:
       ```
       pub fn save_binary(&self, product: &str, spec: &ApiSpec) -> Result<(), ShrugError>
       ```
       - Serialise with `rkyv::to_bytes::<rkyv::rancor::Error>(spec)`
       - Write to `{product}.rkyv` in specs_dir (atomic write pattern like JSON)
       - Save JSON cache too (dual-write for fallback)

    2. Add binary load method:
       ```
       pub fn load_binary(&self, product: &str) -> Result<Option<ApiSpec>, ShrugError>
       ```
       - Read `{product}.rkyv` bytes
       - Access archived data with `rkyv::access::<ArchivedApiSpec, rkyv::rancor::Error>(&bytes)`
       - Deserialise to owned ApiSpec with `rkyv::deserialize::<ApiSpec, rkyv::rancor::Error>(archived)`
       - Return None if file doesn't exist
       - Return Err on corruption (log warning, caller can fall back to JSON)

    3. Update `save()` to also call `save_binary()` (dual-write).

    4. Update `load()` to try binary first, fall back to JSON:
       - Try `load_binary()` first
       - If binary exists and is valid, return it (skip TTL check on binary — TTL is tracked via JSON metadata)
       - If binary fails, fall back to existing JSON `load()`
       - Actually, simpler: check JSON metadata for TTL, then load binary if fresh, fall back to JSON.

    5. Add tests:
       - Binary save/load round-trip
       - Fallback to JSON when no binary exists
       - Corrupted binary falls back to JSON gracefully
  </action>
