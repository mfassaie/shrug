<objective>
## Goal
Add a Markdown → ADF converter for input bodies and JQL shorthand flags for common Jira searches.

## Purpose
Users writing issue descriptions or comments currently must supply raw ADF JSON, which is impractical. A Markdown → ADF converter lets them write natural Markdown that gets transformed into valid ADF before sending to the API. JQL shorthand flags (--project, --assignee, --status) eliminate the need to write raw JQL for the most common search patterns, making Jira queries feel native to the CLI.

## Output
- `src/markdown_to_adf.rs` — Markdown-to-ADF converter module
- `src/jql.rs` — JQL shorthand builder module
- Updated `src/cli.rs` with JQL shorthand flags and `--markdown` flag
- Updated `src/main.rs` to wire both features into the execution pipeline
</objective>

<acceptance_criteria>

## AC-1: Markdown to ADF conversion (common elements)
```gherkin
Given a Markdown string with paragraphs, headings (h1-h6), bold, italic, code spans, bullet lists, ordered lists, code blocks, blockquotes, links, and horizontal rules
When markdown_to_adf() is called
Then a valid ADF JSON document is returned with correct node types, nesting, and marks
```

## AC-2: Markdown to ADF round-trip fidelity
```gherkin
Given a Markdown string converted to ADF
When that ADF is rendered back through the existing render_adf() function
Then the output is semantically equivalent to the original Markdown (headings, lists, code blocks preserved)
```

## AC-3: --markdown flag converts body before sending
```gherkin
Given a user passes --json '{"description": "# Title\n\nSome **bold** text"}' --markdown
When the request is constructed
Then the "description" field value is replaced with valid ADF JSON before sending
```

## AC-4: JQL shorthand flags build correct JQL
```gherkin
Given shorthand flags --project FOO --assignee me --status "In Progress"
When JQL is built from these flags
Then the result is 'project = "FOO" AND assignee = currentUser() AND status = "In Progress"'
```

## AC-5: JQL shorthand combines with raw --jql
```gherkin
Given both --jql "priority = High" and --project FOO are provided
When JQL is built
Then the result is 'project = "FOO" AND priority = High' (shorthand ANDed with raw JQL)
```

## AC-6: JQL shorthand flags only apply to Jira products
```gherkin
Given JQL shorthand flags are passed with a non-Jira product (e.g. confluence)
When the CLI processes the request
Then the shorthand flags are ignored (no error, they are simply unused)
```

  <name>Task 1: Create Markdown → ADF converter module</name>
  <files>src/markdown_to_adf.rs, src/lib.rs, Cargo.toml</files>
  <action>
    Add `pulldown-cmark = "0.12"` to [dependencies] in Cargo.toml.

    Create `src/markdown_to_adf.rs` with:
    - `pub fn markdown_to_adf(markdown: &str) -> serde_json::Value` — main entry point
      - Uses pulldown-cmark to parse Markdown into events
      - Transforms events into ADF node tree
      - Returns `{"type": "doc", "version": 1, "content": [...]}`
    - Support these ADF node types:
      - paragraph, heading (levels 1-6), bulletList, orderedList, listItem
      - codeBlock (with optional language attr), blockquote, rule (horizontal rule)
      - text with marks: strong (bold), em (italic), code (inline code), link (with href attr)
      - hardBreak
    - Event-to-ADF mapping strategy:
      - Maintain a stack of open container nodes (doc, list, blockquote, etc.)
      - On Start tags: push new container onto stack
      - On End tags: pop container and attach to parent
      - On Text/Code/SoftBreak/HardBreak: create inline nodes, attach to current container
    - `pub fn convert_body_markdown(body: &str) -> Result<String, ShrugError>` — convenience function
      - Parses body as JSON
      - Recursively walks JSON object looking for known ADF field keys at any nesting depth <!-- audit-added: handles {"fields":{"description":"..."}} -->
      - Targets known ADF fields by key name: "description", "body", "comment" — at any level of nesting
      - Only converts values that are plain strings (skips values already structured as objects/arrays)
      - Converts matching string values to ADF JSON objects via markdown_to_adf()
      - Returns the modified JSON string
    - Do NOT use unsafe code. Handle malformed Markdown gracefully (pulldown-cmark is permissive).
    - Add comprehensive unit tests:
      - Paragraph text
      - Heading levels 1-6
      - Bold, italic, inline code marks
      - Bullet list (including nested)
      - Ordered list
      - Code block with and without language
      - Blockquote
      - Horizontal rule
      - Links
      - Mixed content (paragraph with bold and links)
      - Round-trip test: markdown_to_adf → render_adf → verify semantic equivalence
      - convert_body_markdown with description field
      - convert_body_markdown with nested field: {"fields": {"description": "# Heading"}} <!-- audit-added -->
      - convert_body_markdown skips values that are already objects (not strings) <!-- audit-added -->
      - Empty/whitespace-only Markdown

    Add `pub mod markdown_to_adf;` to src/lib.rs.
  </action>
  <verify>cargo test markdown_to_adf -- --nocapture && cargo clippy -- -D warnings</verify>
