<objective>
## Goal
Implement shell completion generation and field/user resolution caches for the CLI.

## Purpose
Shell completions make the CLI discoverable without consulting help text. Field resolution (human name → customfield_ID) and user resolution (display name → accountId) remove the friction of looking up Atlassian internal identifiers, which are required by many API operations but not known to users off the top of their heads.

## Output
- `src/completions.rs` — Shell completion generator for bash, zsh, fish, PowerShell
- `src/resolve.rs` — Field name and user display name resolution with file-based caching
- Updated `src/cli.rs` with Completions subcommand args
- Updated `src/main.rs` to handle completions and wire resolution into helpers
</objective>

<task type="auto">
  <name>Task 2: Implement field and user resolution with file-based cache</name>
  <files>src/resolve.rs, src/lib.rs</files>
  <action>
    Create `src/resolve.rs` with:

    **Cache structure:**
    - Cache files stored at `{cache_dir}/resolve/{site_hash}/fields.json` and `users.json`
    - Site hash: first 16 chars of SHA-256 of the site URL (avoids path issues with URLs)
    - Cache file format: `{"updated_at": "ISO8601", "entries": {"human_name": "internal_id", ...}}`
    - TTL: 24 hours (same as spec cache)

    **Field resolution:**
    - `pub struct FieldCache` with `new(cache_dir: PathBuf)` constructor
    - `pub fn resolve(&self, site: &str, human_name: &str) -> Option<String>`
      - Loads `fields.json` for the site
      - Checks TTL (24h), returns None if expired
      - Case-insensitive match against keys
      - Returns the customfield_ID or None
    - `pub fn populate(&self, site: &str, fields: &[(String, String)]) -> Result<(), ShrugError>`
      - Writes name → ID mappings to cache file
      - Used by a future "shrug cache refresh-fields" command or populated on first use

    **User resolution:**
    - `pub struct UserCache` with `new(cache_dir: PathBuf)` constructor
    - `pub fn resolve(&self, site: &str, display_name: &str) -> Option<String>`
      - Loads `users.json` for the site
      - Checks TTL (24h), returns None if expired
      - Case-insensitive match
      - Returns accountId or None
    - `pub fn populate(&self, site: &str, users: &[(String, String)]) -> Result<(), ShrugError>`
      - Writes display_name → accountId mappings

    **Shared cache utilities:**
    - `fn cache_path(base: &Path, site: &str, filename: &str) -> PathBuf`
      - Hashes site URL, creates directory, returns file path
    - `fn is_cache_fresh(path: &Path, ttl_hours: u64) -> bool`
      - Checks file modification time against TTL
    - `fn load_cache(path: &Path) -> Option<HashMap<String, String>>`
      - Reads JSON, checks TTL, returns entries map
    - `fn save_cache(path: &Path, entries: &[(String, String)]) -> Result<(), ShrugError>`
      - Writes JSON with updated_at timestamp
      - Uses write-to-temp-file-then-rename for atomic writes (prevents corrupted cache on interrupted writes) <!-- audit-added -->

    Add `pub mod resolve;` to src/lib.rs.

    Unit tests:
    - FieldCache resolve returns None when no cache exists
    - FieldCache populate then resolve returns correct ID
    - FieldCache resolve is case-insensitive
    - FieldCache returns None for expired cache (mock file time)
    - UserCache resolve returns None when no cache exists
    - UserCache populate then resolve returns correct accountId
    - UserCache resolve is case-insensitive
    - cache_path produces site-scoped paths
    - save_cache and load_cache round-trip
  </action>
  <verify>cargo test resolve -- --nocapture && cargo clippy -- -D warnings</verify>
  <done>AC-3, AC-4, AC-5, AC-6 satisfied: field and user resolution with TTL-based caching, graceful handling of missing/expired cache</done>
