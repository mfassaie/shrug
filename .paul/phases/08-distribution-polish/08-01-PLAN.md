<objective>
## Goal
Set up cargo-dist release automation with GitHub Actions, plus Homebrew tap and Scoop manifest templates for package distribution.

## Purpose
Users need to install shrug without building from source. cargo-dist automates the creation of pre-built binaries for Linux (musl), macOS (Intel + Apple Silicon), and Windows on every GitHub release tag. Homebrew and Scoop manifests let users install via their platform's package manager (`brew install`, `scoop install`).

## Output
- `.github/workflows/release.yml` — cargo-dist release workflow triggered by tags
- `Cargo.toml` — cargo-dist metadata section
- `dist/homebrew/shrug.rb` — Homebrew formula template
- `dist/scoop/shrug.json` — Scoop manifest template
</objective>

<task type="auto">
  <name>Task 1: Configure cargo-dist and create release workflow</name>
  <files>Cargo.toml, .github/workflows/release.yml</files>
  <action>
    Add cargo-dist metadata to Cargo.toml:
    ```toml
    [workspace.metadata.dist]
    cargo-dist-version = "0.27.0"
    ci = "github"
    installers = ["shell", "powershell"]
    targets = [
        "x86_64-unknown-linux-musl",
        "x86_64-apple-darwin",
        "aarch64-apple-darwin",
        "x86_64-pc-windows-msvc",
    ]
    ```

    Create `.github/workflows/release.yml`:
    - Trigger: push tags matching `v*.*.*`
    - Permissions: contents write (for creating releases)
    - Job: use cargo-dist GitHub Action pattern
      - Checkout code
      - Install Rust toolchain
      - Install cargo-dist: `cargo install cargo-dist`
      - Plan: `cargo dist plan`
      - Build: `cargo dist build` for each target
      - Publish: create GitHub Release with artifacts via `gh release create` or cargo-dist's own publish
    - Matrix strategy for cross-platform builds:
      - Linux musl on ubuntu-latest (install musl-tools: `sudo apt-get install -y musl-tools`, add target: `rustup target add x86_64-unknown-linux-musl`) <!-- audit-added: musl toolchain required -->
      - macOS on macos-latest (universal binary or separate x86_64 + aarch64)
      - Windows on windows-latest
    - Use explicit minimal permissions block: `permissions: { contents: write }` with no other permissions granted <!-- audit-added: least-privilege -->
    - Each build uploads its artifact to the release
    - Use actions/upload-artifact and actions/download-artifact for cross-job sharing

    Do NOT modify the existing ci.yml workflow. The release workflow is separate.
  </action>
  <verify>
    Validate YAML syntax: python -c "import yaml; yaml.safe_load(open('.github/workflows/release.yml'))" or manual review.
    Verify Cargo.toml contains [workspace.metadata.dist] section.
  </verify>
  <done>AC-1, AC-2, AC-5 satisfied: release workflow triggers on tags, Cargo.toml has dist metadata, YAML is valid</done>

<task type="auto">
  <name>Task 2: Create Homebrew and Scoop manifest templates</name>
  <files>dist/homebrew/shrug.rb, dist/scoop/shrug.json</files>
  <action>
    Create `dist/homebrew/shrug.rb`:
    ```ruby
    class Shrug < Formula
      desc "A dynamic CLI for Atlassian Cloud"
      homepage "https://github.com/mfassaie/shrug"
      version "VERSION"
      license "MIT"

      on_macos do
        if Hardware::CPU.arm?
          url "https://github.com/mfassaie/shrug/releases/download/vVERSION/shrug-aarch64-apple-darwin.tar.gz"
          sha256 "SHA256"
        else
          url "https://github.com/mfassaie/shrug/releases/download/vVERSION/shrug-x86_64-apple-darwin.tar.gz"
          sha256 "SHA256"
        end
      end

      on_linux do
        url "https://github.com/mfassaie/shrug/releases/download/vVERSION/shrug-x86_64-unknown-linux-musl.tar.gz"
        sha256 "SHA256"
      end

      def install
        bin.install "shrug"
      end

      test do
        assert_match "shrug", shell_output("#{bin}/shrug --version")
      end
    end
    ```
    Note: VERSION and SHA256 are placeholders to be updated by the release process.

    Create `dist/scoop/shrug.json`:
    ```json
    {
        "version": "VERSION",
        "description": "A dynamic CLI for Atlassian Cloud",
        "homepage": "https://github.com/mfassaie/shrug",
        "license": "MIT",
        "architecture": {
            "64bit": {
                "url": "https://github.com/mfassaie/shrug/releases/download/vVERSION/shrug-x86_64-pc-windows-msvc.zip",
                "hash": "SHA256"
            }
        },
        "bin": "shrug.exe",
        "checkver": "github",
        "autoupdate": {
            "architecture": {
                "64bit": {
                    "url": "https://github.com/mfassaie/shrug/releases/download/v$version/shrug-x86_64-pc-windows-msvc.zip"
                }
            }
        }
    }
    ```

    Create parent directories: `dist/homebrew/` and `dist/scoop/`.
  </action>
  <verify>Verify files exist and contain correct structure. JSON syntax: python -c "import json; json.load(open('dist/scoop/shrug.json'))" or manual review.</verify>
  <done>AC-3, AC-4 satisfied: Homebrew formula and Scoop manifest templates exist with correct structure</done>
