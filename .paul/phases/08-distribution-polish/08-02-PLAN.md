## Goal
Add mock-based integration tests that exercise the full request pipeline against recorded API responses.

## Purpose
The existing 386 unit tests verify individual functions in isolation. Integration tests verify that the modules work together end-to-end: spec loading → command routing → request building → HTTP execution → response formatting. Using a mock HTTP server with recorded Jira API responses provides deterministic, fast tests without requiring live API access.

    - Verify no error returned
