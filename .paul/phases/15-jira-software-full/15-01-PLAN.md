<acceptance_criteria>

## AC-1: Board CRUD lifecycle
```gherkin
Given the E2E project exists
When a Jira filter is created for the project
And create-board is called with the filter ID and type "scrum"
Then get-board returns the board with matching name
And get-configuration returns board configuration
And delete-board removes it with exit code 0
```

## AC-2: Sprint lifecycle
```gherkin
Given a scrum board exists (created via Board CRUD)
When create-sprint is called with the board's origin ID
Then get-sprint returns the sprint with matching name
And partially-update-sprint changes the goal
And delete-sprint removes it with exit code 0
```

## AC-3: Board list operations
```gherkin
Given the E2E project has boards
When get-all-boards is called with the project key filter
Then the response contains at least one board entry
```

  <name>Task 1: Create jira_software.rs module and Board CRUD test</name>
  <files>tests/e2e/jira_software.rs, tests/e2e/main.rs</files>
  <action>
    1. Add `mod jira_software;` to `tests/e2e/main.rs` (after the `mod jira;` line).

    2. Create `tests/e2e/jira_software.rs` with module doc comment and imports:
       ```
       //! Jira Software E2E tests: Boards, Sprints, Epics.
       use crate::harness::{self, ShrugRunner};
       ```

    3. Add helper functions following jira.rs pattern:
       - `setup_profile(runner)` and `teardown_profile(runner, name)` — copy from jira.rs
       - `create_filter(runner, project) -> Option<String>` — creates a JQL filter via
         `jira Filters create-filter` with body `{"name":"e2e-jsw-filter-{pid}","jql":"project = {project}"}`,
         returns filter ID. If fails, returns None.
       - `delete_filter(runner, id)` — deletes filter via `jira Filters delete-filter --id {id}`

    4. Implement `test_board_crud_lifecycle()`:
       a. Create profile, get project key
       b. Create a filter for the project via `create_filter()`. If None, skip with eprintln.
       c. Call `jira-software Board create-board` with body:
          `{"name":"e2e-board-{pid}","type":"scrum","filterId":{filterId}}`
          If fails, eprintln and clean up filter. The filterId must be an integer (not quoted).
       d. Extract board `id` from response.
       e. Call `jira-software Board get-board --boardId {id}` and assert success.
          Verify `name` matches.
       f. Call `jira-software Board get-configuration --boardId {id}` and assert success.
       g. Call `jira-software Board delete-board --boardId {id}` and assert exit_code 0.
       h. Clean up: delete_filter, teardown_profile.
       i. Use rate_limit_delay between all calls.
  </action>
  <verify>`cargo test --test e2e test_board_crud_lifecycle` compiles and passes/skips</verify>
  <done>AC-1 satisfied: board create/get/config/delete lifecycle completes</done>
</task>
