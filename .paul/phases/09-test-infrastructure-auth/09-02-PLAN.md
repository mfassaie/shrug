<objective>
## Goal
Add network spec fetching to the SpecLoader so the CLI can download real OpenAPI specs from Atlassian CDN. Add a `shrug cache refresh` command to manually trigger spec downloads. This unblocks all E2E testing.

## Purpose
The v0.1 MVP bundled placeholder specs with zero operations. Without real specs, the CLI cannot discover or execute any API operations. Every E2E test in Phases 9-12 depends on this.

## Output
- SpecLoader gains a network fetch tier: cache → network → bundled
- `shrug cache refresh` command downloads all specs
- `shrug cache refresh --product jira` downloads a single spec
</objective>

<task type="auto">
  <name>Task 1: Add network fetch to SpecLoader and cache refresh CLI command</name>
  <files>src/spec/registry.rs, src/main.rs, src/cli.rs</files>
  <action>
    **src/spec/registry.rs — Add fetch_spec method to SpecLoader:**
    - Add `fetch_spec(&self, product: &Product) -> Result<ApiSpec, ShrugError>` method:
      - Use `reqwest::blocking::Client` (already a dependency) to GET the spec URL from `product.info().spec_url`
      - Set timeout to 30 seconds, user-agent to "shrug/{version}"
      - Validate HTTP response status: if not 200, return a ShrugError::SpecError with the status code and URL. Do not attempt to parse non-200 responses. <!-- audit-added -->
      - On 200: parse the response body with `parse_spec()` (V3) or `parse_swagger()` (Swagger 2.0 for BitBucket)
      - Save parsed spec to cache via `self.cache.save()`
      - Return the parsed spec
    - Print progress before each download: `eprintln!("Fetching {display_name}...")` so the user knows what's happening (some specs are 2.47MB and take seconds). <!-- audit-added -->
    - Update `load()` method to add network tier:
      ```
      Tier 1: cache (TTL-based) → return if hit
      Tier 2: network fetch → return if successful, save to cache
      Tier 3: bundled fallback → return as last resort
      ```
    - Add `pub fn refresh(&self, product: &Product) -> Result<ApiSpec, ShrugError>` that always fetches from network (ignoring cache), saves to cache, and returns the spec
    - Add `pub fn refresh_all(&self) -> Vec<(Product, Result<ApiSpec, ShrugError>)>` that refreshes all 5 products, returning results for each
    - Use `Product::all()` or the existing `ALL_PRODUCTS` array to iterate

    **src/cli.rs — Add CacheCommands enum:**
    - Add `CacheCommands` enum with:
      - `Refresh { product: Option<String> }` — refresh one or all specs
    - Update the `Cache` variant in `Commands` to use `CacheCommands` (currently it just takes trailing args)

    **src/main.rs — Handle cache refresh command:**
    - Match on `Commands::Cache { command }`:
      - `CacheCommands::Refresh { product: None }` → call `loader.refresh_all()`, print results
      - `CacheCommands::Refresh { product: Some(name) }` → resolve product name, call `loader.refresh()`, print result
    - Print: "Refreshed {product}: {operations} operations ({size})" for each spec
    - Print total specs refreshed

    Avoid: Using async reqwest — the existing codebase uses `reqwest::blocking`.
    Avoid: Breaking existing `shrug cache` behaviour — the trailing args pattern should still work for future cache subcommands.
    Avoid: Making network fetch mandatory — if network fails, fall back gracefully.
