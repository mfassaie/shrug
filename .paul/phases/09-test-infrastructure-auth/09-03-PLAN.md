<acceptance_criteria>

## AC-1: Profile Create and List
```gherkin
Given no profiles exist (clean state)
When the user runs `shrug profile create --name test1 --site site.atlassian.net --email a@b.com`
Then the profile is created and `shrug profile list` shows it
```

## AC-2: Profile Show and Use
```gherkin
Given two profiles exist (test1, test2)
When the user runs `shrug profile use --name test2`
Then `shrug profile show --name test2` displays correct details and test2 is the default
```

## AC-3: Profile Delete
```gherkin
Given a profile "test1" exists
When the user runs `shrug profile delete --name test1`
Then `shrug profile list` no longer shows test1
```

## AC-4: Environment Variable Auth
```gherkin
Given a profile exists and SHRUG_API_TOKEN is set
When the user runs a Jira search operation
Then the request authenticates successfully using the env var token
```

## AC-5: First-Run Detection
```gherkin
Given no profiles exist and no env vars are set
When the user runs `shrug jira --help`
Then the command succeeds (help doesn't require auth)
```

## AC-6: First-Run Error on API Call
```gherkin
Given no profiles exist
When the user runs an API operation (e.g., jira search)
Then exit code 3 (auth error) is returned with message mentioning `shrug auth setup`
```

## AC-7: Profile Flag Override
```gherkin
Given two profiles exist (default: test1, non-default: test2)
When the user runs a command with `--profile test2`
Then the test2 profile's credentials are used (not test1's)
```

## AC-8: Auth Status Command
```gherkin
Given a profile exists with env var token set
When the user runs `shrug auth status --profile test1`
Then the output reports token status for that profile
```
