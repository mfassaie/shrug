<task type="auto">
  <name>Task 1: Implement spec analysis utilities</name>
  <files>src/spec/analysis.rs, src/spec/mod.rs</files>
  <action>
    Create src/spec/analysis.rs with:

    1. **URL building:**
       ```
       build_url(server_url: Option<&str>, path: &str, path_params: &HashMap<String, String>) -> Result<String, ShrugError>
       ```
       - Substitute {param_name} placeholders in path template with provided values
       - Prepend server_url if present (handle trailing slash dedup)
       - Error if a path placeholder has no matching param value
       - Ignore extra params not in the path template
       - Percent-encode param values for path segments (not full URL encoding) <!-- audit-modified -->
         Use percent_encoding crate's PATH_SEGMENT encoding or manual implementation
         This preserves characters valid in path segments while encoding unsafe ones
       - If server_url contains {variable} templates (e.g., "{baseUrl}"), strip them or leave as-is with warning <!-- audit-added -->

    2. **Query string building:**
       ```
       build_query_string(params: &[(String, String)]) -> String
       ```
       - Build "key=value&key2=value2" from provided params
       - URL-encode keys and values
       - Return empty string if no params

    3. **Pagination detection:**
       ```
       detect_pagination(operation: &Operation) -> Option<PaginationStyle>
       ```
       PaginationStyle enum:
       - Offset { start_param: String, limit_param: String } — startAt/maxResults pattern (Jira)
       - Page { page_param: String, size_param: String } — page/pagelen pattern (BitBucket)
       - Cursor { cursor_param: String } — cursor/after pattern

       Detection logic (check query param names):
       - If has "startAt" + "maxResults" → Offset
       - If has "start" + "limit" → Offset (Confluence variant)
       - If has "page" + "pagelen" → Page (BitBucket)
       - If has "cursor" or "after" query param → Cursor
       - Otherwise → None

    4. **Path template validation:**
       ```
       validate_path_params(operation: &Operation) -> Vec<String>
       ```
       - Extract {placeholder} names from operation.path
       - Check each has a matching parameter with location=Path
       - Return list of missing param names (empty = valid)

    5. **Parameter helpers:**
       ```
       path_params(operation: &Operation) -> Vec<&Parameter>
       query_params(operation: &Operation) -> Vec<&Parameter>
       required_params(operation: &Operation) -> Vec<&Parameter>
       ```

    Include unit tests:
    - URL building: basic substitution, multiple params, missing param error, extra params ignored, percent encoding of path segments, no server_url, trailing slash dedup, server_url with {variable} template (audit-added)
    - Query string: basic building, URL encoding, empty params
    - Pagination: offset detection (Jira), page detection (BitBucket), cursor detection, none detection, Confluence start/limit variant
    - Path validation: valid operation, missing path param detected
    - Parameter helpers: filter by location, filter required

    Wire up in src/spec/mod.rs:
    - Add `pub mod analysis;`
    - Re-export: `pub use analysis::{build_url, detect_pagination, PaginationStyle};`
  </action>
  <verify>cargo test spec::analysis -- --nocapture && cargo clippy -- -D warnings</verify>
  <done>AC-1, AC-2, AC-3 satisfied</done>
</task>
