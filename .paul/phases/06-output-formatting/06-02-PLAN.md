<objective>
## Goal
Add --fields column selection for table/CSV output, pager integration for long output, and ADF (Atlassian Document Format) terminal rendering so rich-text API responses are human-readable.

## Purpose
Atlassian API responses often contain dozens of fields, making table output noisy. --fields lets users pick the columns they care about. Long responses scroll past the terminal without a pager. Description/body fields contain ADF JSON which is unreadable without rendering. These three features complete the output layer.

## Output
- --fields flag for column filtering in table and CSV output
- Pager integration ($PAGER, --no-pager)
- New src/adf.rs module for ADF → terminal text conversion
- ADF rendering integrated into output formatters
</objective>

<acceptance_criteria>

## AC-1: --fields filters table columns
```gherkin
Given a JSON response with multiple fields
When --fields key,summary,status is specified
Then only those columns appear in table output
And column order matches the --fields argument order
```

## AC-2: --fields filters CSV columns
```gherkin
Given a JSON response with multiple fields
When --fields is specified with --output csv
Then only those columns appear in CSV output with matching header row
```

## AC-3: --fields ignored for JSON/YAML/plain
```gherkin
Given --fields is specified with --output json
When the response is formatted
Then the full JSON is output unchanged (--fields only affects table and CSV)
```

## AC-4: Pager activated for long TTY output
```gherkin
Given stdout is a TTY and output exceeds terminal height
When the formatted output is displayed
Then it is piped through $PAGER (defaulting to "less -R")
And ANSI colour codes are preserved
```

## AC-5: --no-pager disables pager
```gherkin
Given --no-pager is specified
When the output is displayed
Then it is printed directly to stdout regardless of length
```

## AC-6: ADF rendered as terminal text
```gherkin
Given a JSON response field containing ADF content (type: "doc")
When table or plain output is rendered
Then ADF paragraphs become plain text with line breaks
And headings are prefixed with # markers
And bullet lists use - markers
And code blocks are indented with 4 spaces
And bold/italic text uses ANSI formatting when colour is enabled
```

## AC-7: ADF in JSON/YAML output left unchanged
```gherkin
Given --output json or --output yaml
When a response contains ADF content
Then the ADF JSON is included as-is (no rendering)
```

    1. Create `src/adf.rs` with:

       a. `pub fn render_adf(adf: &serde_json::Value, color_enabled: bool) -> String`
          - If value is not an object with "type": "doc", return the raw JSON string
          - Walk the "content" array recursively
          - Render each node type:

       b. Node type rendering:
          - `paragraph` → render inline content, append double newline
          - `heading` → prefix with # (level 1), ## (level 2), etc. Append double newline
          - `bulletList` → render each listItem with "- " prefix
          - `orderedList` → render each listItem with "1. ", "2. " numbering
          - `listItem` → render content with indentation
          - `codeBlock` → indent each line with 4 spaces, append newline
          - `blockquote` → prefix each line with "> "
          - `rule` → render as "---\n"
          - `text` → return text content, apply marks if colour enabled:
            - `strong` → ANSI bold (\x1b[1m...\x1b[0m)
            - `em` → ANSI italic (\x1b[3m...\x1b[0m)
            - `code` → ANSI dim (\x1b[2m...\x1b[0m)
            - `link` → append URL in parentheses: "text (url)"
          - `hardBreak` → newline
          - `mention` → "@displayName" or "@id"
          - `emoji` → shortName or fallback text
          - Unknown types → skip silently (forward-compatible)

    2. Add `pub mod adf;` to src/lib.rs

    3. Integrate ADF rendering into output.rs:
       a. In `truncate_value()`: detect ADF content (object with "type": "doc" and "content" array)

       - render_adf with text marks produces ANSI when colour enabled
       - render_adf with text marks produces no ANSI when colour disabled

       - render_adf with unknown node types skips gracefully
       - output integration: ADF in table cell is rendered as text
