<objective>
## Goal
Create output formatters (JSON, table, YAML, CSV, plain) with TTY detection so the executor renders API responses in the user's chosen format instead of dumping raw JSON.

## Purpose
Currently `println!("{}", body)` dumps raw JSON regardless of `--output` flag. Users need readable table output in terminals and machine-parseable JSON/YAML/CSV for pipelines. TTY detection should auto-select table for interactive use and JSON for piped output.

## Output
- New `src/output.rs` module with format_response() function
- 5 formatters: JSON (pretty-printed), table, YAML, CSV, plain text
- TTY detection for auto-format selection
- NO_COLOR support
- Executor integration: format_response() replaces raw println
</objective>

## AC-1: JSON output is pretty-printed
```gherkin
Given a JSON API response
When --output json is specified (or auto-selected for piped output)
Then the response is printed as pretty-printed JSON with 2-space indentation
```

## AC-2: Table output renders key-value or columnar data
```gherkin
Given a JSON API response containing an object or array of objects
When --output table is specified (or auto-selected for TTY)
Then the response is rendered as a formatted terminal table with column headers
And nested objects/arrays are shown as truncated JSON strings in cells
```

## AC-3: YAML output renders valid YAML
```gherkin
Given a JSON API response
When --output yaml is specified
Then the response is printed as valid YAML
```

## AC-4: CSV output renders with headers
```gherkin
Given a JSON API response containing an array of objects
When --output csv is specified
Then the response is printed as CSV with a header row from object keys
And scalar values are unquoted unless they contain commas or newlines
```

## AC-5: Plain text output renders flat key-value pairs
```gherkin
Given a JSON API response
When --output plain is specified
Then top-level scalar fields are rendered as "key: value" lines
And arrays/objects are rendered as compact JSON
```

## AC-6: TTY auto-detection selects correct format
```gherkin
Given no explicit --output flag (default auto behaviour)
When stdout is a TTY
Then table format is used
When stdout is not a TTY (piped)
Then JSON format is used
```

## AC-7: NO_COLOR respected
```gherkin
Given the NO_COLOR environment variable is set
When table output is rendered
Then no ANSI colour codes are present in the output
```

## AC-8: Pagination output formatted per page
```gherkin
Given a paginated response (--page-all)
When output format is table
Then each page is rendered as a separate table (consistent with print-as-you-go)
```

<task type="auto">
  <name>Task 1: Add dependencies and create output formatter module</name>
  <files>Cargo.toml, src/output.rs, src/lib.rs</files>
  <action>
    1. Add dependencies to Cargo.toml:
       - `comfy-table = "7"` — terminal table rendering
       - `serde_yaml = "0.9"` — YAML serialisation
       - `csv = "1"` — CSV writing
       - `is-terminal = "0.4"` — TTY detection (replaces deprecated atty)

    2. Create `src/output.rs` with:

       a. `format_response(body: &str, format: &OutputFormat, is_tty: bool, color_enabled: bool) -> String`
          - Attempts to parse body as serde_json::Value
          - If JSON parsing fails, return the raw body unchanged (Atlassian APIs may return HTML error pages or plain text) <!-- audit-added -->
          - Dispatches to format-specific renderer
          - If format is Table and is_tty is false (auto mode), fall back to JSON
          - Returns formatted string ready for println

       b. `format_json(json: &serde_json::Value) -> String`
          - serde_json::to_string_pretty with 2-space indent

       c. `format_table(json: &serde_json::Value, color_enabled: bool) -> String`
          - If json is an object: render as 2-column key/value table
          - If json contains a known results array (issues, values, results): render array items as columnar table
          - If json is a top-level array: render as columnar table
          - Column headers from object keys, truncate wide values at 60 chars
          - Use comfy-table for rendering
          - Disable ANSI styling when color_enabled is false

       d. `format_yaml(json: &serde_json::Value) -> String`
          - serde_yaml::to_string

       e. `format_csv(json: &serde_json::Value) -> String`
          - Extract array of objects (from known fields or top-level)
          - Write CSV with headers from first object's keys, sorted alphabetically for deterministic output <!-- audit-added -->
          - Scalar values as-is, nested objects as compact JSON
          - If not an array of objects, fall back to JSON

       f. `format_plain(json: &serde_json::Value) -> String`
          - Top-level object: "key: value" per line
          - Top-level array: one item per line
          - Nested values: compact JSON inline

       g. `resolve_format(explicit: &OutputFormat, is_tty: bool) -> OutputFormat`
          - If Table and not TTY, return Json (auto-detection)
          - Otherwise return the explicit format

       h. `should_use_color(choice: &ColorChoice, is_tty: bool) -> bool`
          - Auto: is_tty && env NO_COLOR not set
          - Always: true
          - Never: false

    3. Add `pub mod output;` to `src/lib.rs`.

    4. Unit tests:
       - format_json produces pretty-printed output
       - format_table renders object as key/value table
       - format_table renders array as columnar table
       - format_table extracts "issues" array from wrapper object
       - format_yaml produces valid YAML
       - format_csv produces header row and data rows
       - format_csv falls back to JSON for non-array input
       - format_plain renders key: value pairs
       - resolve_format returns Json when Table + not TTY
       - resolve_format returns Table when Table + TTY
       - should_use_color respects Never/Always/Auto
       - format_response returns raw body when input is not valid JSON <!-- audit-added -->
       - format_csv produces alphabetically sorted column headers <!-- audit-added -->

    Avoid: Over-engineering table rendering. Do not attempt column width auto-sizing based on terminal width in this plan. Simple fixed truncation at 60 chars is sufficient. Do not add colour to JSON/YAML/CSV outputs.
  </action>
