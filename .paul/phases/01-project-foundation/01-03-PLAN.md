<acceptance_criteria>

## AC-1: All diagnostic output goes to stderr
```gherkin
Given the tracing subscriber is initialized
When tracing events (info, debug, trace) are emitted
Then all output is written to stderr, not stdout
And stdout is reserved exclusively for command output data
```

## AC-2: Verbosity flags control log level
```gherkin
Given the CLI is invoked with verbosity flags
When -v is passed, tracing shows info-level messages
And when -vv is passed, tracing shows debug-level messages
And when --trace is passed, tracing shows trace-level messages
And when no flag is passed, only warn-level messages appear
```

## AC-3: Color-aware log formatting
```gherkin
Given the --color flag is set
When tracing output is rendered
Then ANSI colors are used when color=auto (TTY) or color=always
And ANSI colors are suppressed when color=never or NO_COLOR env var is set
```

## AC-4: Debug-level config dump at startup
```gherkin
Given the user runs shrug with -vv or higher
When config is loaded
Then the resolved ShrugConfig is logged at debug level
And sensitive fields (if any in future) are not exposed
```

## AC-5: Graceful Ctrl+C handling
```gherkin
Given shrug is running a long operation
When the user presses Ctrl+C
Then the process exits with exit code 130 (standard SIGINT convention)
And no panic or ugly stack trace is shown
```

       - Configure tracing_subscriber::fmt():

         - `.with_ansi(should_use_color(color))` — respect color setting

         - `.with_env_filter(...)` — RUST_LOG env var overrides, fallback to computed level
       - `should_use_color(color: &ColorChoice) -> bool` helper:
         - Auto: check if stderr is a TTY AND NO_COLOR env var is not set
         - Always: true
         - Never: false
       - Use `std::io::IsTerminal` trait (stable since Rust 1.70) for TTY detection
       <!-- audit-added: Make should_use_color testable -->
       - Make `should_use_color` pub(crate) so it can be unit tested

       <!-- audit-added: Color logic has branching that must be tested -->

       - `should_use_color(&ColorChoice::Never)` returns false
       - `should_use_color(&ColorChoice::Auto)` respects NO_COLOR env var (set NO_COLOR=1, assert false; unset, result depends on TTY)

    5. Update src/lib.rs to add `pub mod logging;`

