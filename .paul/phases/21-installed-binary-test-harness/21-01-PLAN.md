<objective>
## Goal
Create a new `tests/smoke/` test target with a harness that runs tests against an installed `shrug.exe` binary found on PATH (or via `SHRUG_E2E_BINARY` env var), supporting both offline tests (no credentials) and online tests (gated by E2E env vars).

## Purpose
The existing E2E tests use `assert_cmd::Command::cargo_bin("shrug")` which tests the build output, not the installed binary. This harness tests the actual user experience: binary on PATH, Windows ANSI support, real config/cache paths. Offline tests can run without Atlassian credentials, catching CLI regressions in CI without needing live API access.

## Output
- `tests/smoke/main.rs` — Test module root
- `tests/smoke/harness.rs` — SmokeRunner (PATH-based binary discovery), SmokeConfig, RunResult, skip macros
- `tests/smoke/fixtures.rs` — ResourceTracker adapted for SmokeRunner, create/delete helpers
- Two validation tests proving the harness works (--version, --help)
</objective>

    Create `tests/smoke/harness.rs` with:

    **SmokeConfig** struct:
    - `binary_path: PathBuf` — resolved path to shrug binary
    - `timeout_secs: u64` — from SHRUG_E2E_TIMEOUT_SECS or default 30

    **E2eConfig** struct (reuse exact same structure as tests/e2e/harness.rs):
    - site, email, token, jira_project, confluence_space, delay_ms, timeout_secs
    - `from_env()` method returning Option<Self>

    **SmokeConfig::try_resolve()** constructor returning `Option<SmokeConfig>`: <!-- audit-added: try-pattern -->
    1. Check SHRUG_E2E_BINARY env var — if set, use that path directly
    2. Otherwise, use `which::which("shrug")` to find on PATH
    3. Return None if neither works (do NOT panic — skip macros depend on graceful failure)
    - Add `which` crate as dev-dependency in Cargo.toml

    **SmokeConfig::resolve()** convenience wrapper:
    - Calls try_resolve() and panics with clear message if None:
      "shrug not found on PATH. Set SHRUG_E2E_BINARY or install shrug."
    - Only used in contexts where binary absence is a hard failure (not in skip macros)

    **RunResult** struct (same as existing):
    - stdout: String, stderr: String, exit_code: i32, json: Option<Value>
    - assert_success(), assert_exit_code(), assert_stdout_contains(), assert_stderr_contains(), json_field()
    - Add assert_stderr_contains() — not in existing harness but needed for error message tests in later phases

    **SmokeRunner** struct:
    - Holds SmokeConfig
    - `new(config: SmokeConfig)` — offline runner (no credentials)
    - `with_e2e(config: SmokeConfig, e2e: E2eConfig)` — online runner (injects credentials as env vars)
    - `run(&self, args: &[&str]) -> RunResult` — executes binary_path with args, captures output
      Uses assert_cmd::Command::new(binary_path) (NOT cargo_bin, NOT raw std::process::Command)
      For offline mode (no E2eConfig): call .env_remove("SHRUG_SITE"), .env_remove("SHRUG_EMAIL"),
        .env_remove("SHRUG_API_TOKEN") to prevent credential leakage from test environment <!-- audit-added -->
      For online mode: sets SHRUG_SITE, SHRUG_EMAIL, SHRUG_API_TOKEN from E2eConfig
      Uses assert_cmd's .timeout() for timeout handling
    - `run_json(&self, args: &[&str]) -> RunResult` — prepends --output json, parses JSON
    - `run_json_with_body(&self, body: &str, args: &[&str]) -> RunResult`
    - `run_with_body(&self, body: &str, args: &[&str]) -> RunResult`
    - `config() -> &SmokeConfig`
    - `e2e_config() -> Option<&E2eConfig>`

    **skip_unless_binary!() macro:**
    - Calls SmokeConfig::try_resolve() (not resolve() — must not panic) <!-- audit-added -->
    - Returns SmokeConfig if binary found, otherwise skips with diagnostic message
    - Used for offline tests

    **skip_unless_e2e!() macro:**
    - Calls SmokeConfig::try_resolve() first, skips if binary not found <!-- audit-added -->
    - Then checks E2eConfig::from_env(), skips if credentials missing
    - Returns (SmokeConfig, E2eConfig) tuple if both available

    **rate_limit_delay(config: &E2eConfig):**
    - Same as existing

    Use `assert_cmd::Command::new(binary_path)` for all execution. <!-- audit-added: removed wait_timeout ambiguity -->
    assert_cmd::Command::new() works with any binary path and provides .timeout() method.
    Do NOT add wait_timeout crate — assert_cmd handles timeout already.
    Do NOT use raw std::process::Command — assert_cmd provides better output capture and timeout.
