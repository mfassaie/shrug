<objective>
## Goal
Add `confluence +create` helper command for creating Confluence pages from Markdown files or inline content. Convert Markdown to Confluence storage format (XHTML) for the page body.

## Purpose
Mirrors the UX pattern of `jira +create` for Confluence. Users can create pages from the CLI without manually constructing JSON bodies or converting Markdown to storage format.

## Output
- `src/markdown_to_storage.rs` — Markdown → Confluence storage format converter
- Updated `src/helpers.rs` — `confluence +create` helper with `--space`, `--title`, `--file`, `--parent-id`, `--body`
- Dispatch extended to support Confluence product
</objective>

```gherkin
And the body is converted from Markdown to storage format
And the response page ID is printed
```

    1. Create `src/markdown_to_storage.rs` using pulldown-cmark (already a dependency).

    2. Implement `pub fn markdown_to_storage(markdown: &str) -> String`:
       - Parse Markdown with pulldown-cmark
       - Convert events to Confluence storage format XHTML:
         - Paragraphs → `<p>...</p>`
         - Headings → `<h1>` through `<h6>`
         - Bold → `<strong>`
         - Italic → `<em>`
         - Strikethrough → `<del>`
         - Code spans → `<code>`
         - Code blocks → `<ac:structured-macro ac:name="code"><ac:plain-text-body><![CDATA[...]]></ac:plain-text-body></ac:structured-macro>`
         - With language: add `<ac:parameter ac:name="language">lang</ac:parameter>`
         - Bullet lists → `<ul><li>...</li></ul>`
         - Ordered lists → `<ol><li>...</li></ol>`
         - Links → `<a href="url">text</a>`
         - Blockquotes → `<blockquote>...</blockquote>`
         - Horizontal rules → `<hr />`
         - Hard breaks → `<br />`
         - Images → `<ac:image><ri:url ri:value="url" /></ac:image>`

    3. Add `pub mod markdown_to_storage;` to lib.rs.

    4. Add unit tests:
       - Paragraph conversion
       - Heading levels (h1-h3)
       - Bold, italic, code spans
       - Bullet and ordered lists
       - Code blocks (with and without language)
       - Links
       - Mixed content

    1. Extend `dispatch_helper` to support Confluence:
       - Remove or modify the Jira-only check on line 49
       - For Confluence: route "create" to `helper_confluence_create`
       - Keep existing Jira helpers unchanged
       - Update AVAILABLE_HELPERS or add CONFLUENCE_HELPERS constant

    2. Implement `helper_confluence_create(args, client, credential, dry_run)`:
       - Parse args: --space (required), --title (required), --body (optional), --file (optional), --parent-id (optional)
       - Require either --body or --file (error if neither)
       - If --file: read file contents with `std::fs::read_to_string`
       - Convert Markdown content to storage format via `markdown_to_storage::markdown_to_storage`
       - Build request body:
         ```json
         {
           "spaceId": "<space_id>",
           "title": "<title>",
           "body": {
             "representation": "storage",
             "value": "<converted storage format>"
           },
           "status": "current"
         }
         ```
       - Note: --space accepts a space key. Need to resolve it to a space ID first.
         The Confluence v2 API uses space IDs (numeric), not keys.
         Do a GET /wiki/api/v2/spaces?keys={key} to resolve key → ID.
       - If --parent-id provided, add `"parentId": "<parent_id>"` to the body.
       - POST to /wiki/api/v2/pages
       - On success: print page ID and URL

    3. Add `resolve_space_id(client, credential, space_key)` helper:
       - GET /wiki/api/v2/spaces?keys={key}
       - Extract first result's ID
       - Error if not found

    4. Add unit tests for arg parsing edge cases.
  </action>
